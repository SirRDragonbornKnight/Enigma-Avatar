## ⚠️ KEY LESSONS (read before touching motions / rig posing — saves hours)
- **A limb's local "pitch" is NOT flexion.** On makiro the upper-arm's local-X is ABDUCTION (spreads sideways), the forearm's is the elbow, legs vary again. Posing limbs on a fixed local axis is WRONG per-rig. Use `flex(role, ang)` in procedural.js — it rotates each limb about the BODY's left→right hinge axis (derived per-bone at build, like the eye fix), so forward/back flexion is correct on ANY rig and auto-mirrors L/R. TORSO/neck/head DO flex on local pitch (proven by emote signs) — leave those on `pose()`.
- **Front-view FORESHORTENING hides sagittal motion.** A forward crouch/tuck collapses to a near-straight SILHOUETTE head-on (you'll swear the legs aren't bending — they ARE; verify with bus `query joints` → knee 177°→~110° during a jump, or rotate her 90° to see the profile). The user sees her FRONT-ON, so motions need cues that read head-on: the jump now has a vertical SQUASH-AND-STRETCH dip (sink→spring→absorb) + hair physics, not just leg bend. Apply the same thinking to future motions.
- **DIAGNOSTIC bus queries (use these — they'd have saved this whole session):** `query roles` (role→actual bone name + flex axes), `query joints` (live knee/elbow angle, 180=straight + whether the embedded clip is overriding), `query eyegaze` (each eye's world gaze vector — x flips L↔R only if horizontal eye-look works). All in avatar.js `answerQuery`.
